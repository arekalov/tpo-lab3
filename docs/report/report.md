# Прецеденты использования — StackOverflow

| **Прецедент: Просмотр главной страницы с вопросами** |
|---|
| **ID:** 1 |
| **Краткое описание:** Ознакомиться со списком актуальных вопросов |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** нет |

**Основной поток:**
1. Пользователь переходит на stackoverflow.com.
2. Система отображает список актуальных вопросов с заголовками.
3. Система отображает навигационную панель (поиск, теги, меню).
4. Пользователь прокручивает список и выбирает интересующий вопрос.

---

| **Прецедент: Поиск вопроса по ключевым словам** |
|---|
| **ID:** 2 |
| **Краткое описание:** Найти релевантные вопросы и ответы по ключевым словам |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** нет |

**Основной поток:**
1. Пользователь вводит ключевые слова в строку поиска.
2. Пользователь нажимает Enter или кнопку поиска.
3. Система формирует запрос к поисковому индексу.
4. Система отображает список найденных вопросов с заголовками.
5. Пользователь просматривает результаты и выбирает интересующий вопрос.
6. Система открывает страницу выбранного вопроса.

**Альтернативный поток (ничего не найдено):**

4а. Система не находит совпадений и отображает сообщение «No results found».

---

| **Прецедент: Просмотр карточки вопроса** |
|---|
| **ID:** 3 |
| **Краткое описание:** Прочитать полный текст вопроса, ответы, теги и статус решения |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** нет |

**Основной поток:**
1. Пользователь нажимает на заголовок вопроса в списке.
2. Система открывает страницу вопроса `/questions/{id}`.
3. Система отображает заголовок, тело вопроса, теги.
4. Система отображает список ответов и статус принятого ответа.
5. Пользователь читает вопрос и ответы.

---

| **Прецедент: Регистрация/вход в аккаунт** |
|---|
| **ID:** 4 |
| **Краткое описание:** Получить доступ к функциям участия в сообществе |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** Пользователь не авторизован |

**Основной поток:**
1. Пользователь нажимает ссылку «Log in» в шапке сайта.
2. Система отображает форму входа с полями email и password.
3. Пользователь вводит корректный email в поле «Email».
4. Пользователь вводит корректный пароль в поле «Password».
5. Пользователь нажимает кнопку «Log in».
6. Система проверяет учётные данные и создаёт сессию.
7. Система перенаправляет на главную страницу с именем пользователя.

**Альтернативный поток (неверные учётные данные):**

6а. Система обнаруживает несоответствие email/пароля и отображает сообщение об ошибке.

---

| **Прецедент: Создание нового вопроса** |
|---|
| **ID:** 5 |
| **Краткое описание:** Опубликовать новый вопрос с заголовком, текстом и тегами |
| **Главный актер:** Авторизованный пользователь |
| **Второстепенные актеры:** нет |
| **Предусловия:** Пользователь авторизован («include» UC-04) |

**Основной поток:**
1. Пользователь нажимает кнопку «Ask Question».
2. Система проверяет авторизацию пользователя («include» UC-04).
3. Система открывает форму создания вопроса.
4. Пользователь вводит заголовок вопроса в поле «Title».
5. Пользователь вводит тело вопроса в редактор.
6. Пользователь добавляет один или несколько тегов.
7. Пользователь нажимает «Review your question».
8. Система выполняет предварительную проверку полей.
9. Пользователь нажимает «Post your question».
10. Система сохраняет вопрос и перенаправляет на его страницу.

**Альтернативный поток (незаполненный заголовок):**

8а. Система обнаруживает пустое поле «Title» и отображает ошибку валидации.

---

| **Прецедент: Добавление ответа** |
|---|
| **ID:** 6 |
| **Краткое описание:** Дать ответ на опубликованный вопрос |
| **Главный актер:** Авторизованный пользователь |
| **Второстепенные актеры:** нет |
| **Предусловия:** Пользователь просматривает вопрос («extend» UC-03), пользователь авторизован («include» UC-04) |

**Основной поток:**
1. Пользователь открывает страницу вопроса («extend» UC-03).
2. Система проверяет авторизацию пользователя («include» UC-04).
3. Пользователь прокручивает страницу до редактора ответа.
4. Пользователь вводит текст ответа в редактор.
5. Пользователь нажимает кнопку «Post Your Answer».
6. Система сохраняет ответ и отображает его на странице вопроса.

**Альтернативный поток (пользователь не авторизован):**

2а. Система перенаправляет на страницу входа («include» UC-04).

---

| **Прецедент: Голосование за вопрос/ответ** |
|---|
| **ID:** 7 |
| **Краткое описание:** Повысить или понизить рейтинг контента |
| **Главный актер:** Авторизованный пользователь |
| **Второстепенные актеры:** нет |
| **Предусловия:** Пользователь просматривает вопрос («extend» UC-03), пользователь авторизован («include» UC-04) |

**Основной поток:**
1. Пользователь открывает страницу вопроса («extend» UC-03).
2. Система проверяет авторизацию пользователя («include» UC-04).
3. Пользователь нажимает кнопку ▲ (upvote) или ▼ (downvote) рядом с постом.
4. Система проверяет право на голосование (репутация ≥ 15 для upvote).
5. Система записывает голос и обновляет счётчик.
6. Система визуально выделяет нажатую кнопку.

**Альтернативный поток (недостаточно репутации):**

4а. Система отображает всплывающее сообщение о недостаточной репутации.

---

| **Прецедент: Просмотр тегов и переход по тегу** |
|---|
| **ID:** 8 |
| **Краткое описание:** Найти вопросы по тематике через теги |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** нет |

**Основной поток:**
1. Пользователь нажимает на тег (например, `kotlin`) под вопросом.
2. Система перенаправляет на `/questions/tagged/kotlin`.
3. Система отображает список вопросов с данным тегом.
4. Пользователь просматривает список и выбирает интересующий вопрос.
5. При необходимости пользователь переходит на следующую страницу.

---

| **Прецедент: Добавление вопроса в избранное** |
|---|
| **ID:** 9 |
| **Краткое описание:** Сохранить вопрос для быстрого доступа |
| **Главный актер:** Авторизованный пользователь |
| **Второстепенные актеры:** нет |
| **Предусловия:** Пользователь просматривает вопрос («extend» UC-03), пользователь авторизован («include» UC-04) |

**Основной поток:**
1. Пользователь открывает страницу вопроса («extend» UC-03).
2. Система проверяет авторизацию пользователя («include» UC-04).
3. Пользователь нажимает кнопку «Save» рядом с вопросом.
4. Система добавляет вопрос в список сохранённых пользователя.
5. Система визуально выделяет кнопку «Save» как активную.

**Альтернативный поток (пользователь не авторизован):**

2а. Система отображает предложение войти в аккаунт («include» UC-04).

---

| **Прецедент: Просмотр сохранённых вопросов** |
|---|
| **ID:** 10 |
| **Краткое описание:** Найти ранее сохранённые вопросы |
| **Главный актер:** Авторизованный пользователь |
| **Второстепенные актеры:** нет |
| **Предусловия:** Пользователь авторизован («include» UC-04), ранее добавлял вопросы в избранное (UC-09) |

**Основной поток:**
1. Пользователь переходит на страницу своего профиля.
2. Пользователь нажимает вкладку «Saves».
3. Система отображает список сохранённых вопросов.
4. Пользователь выбирает вопрос из списка.
